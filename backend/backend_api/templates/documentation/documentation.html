Description:
Fetches list of products. See notes on individual parameters below.

Response
Each product object in the response array will have the following parameters:

  • id (integer): The unique identifier of the product.
  • name (string): The name of the product.
  • description (string): The description of the product.
  • price (number): The price of the product.
  • oldPrice (number): The previous price of the product (if applicable).
  • isApple (boolean): Indicates whether the product is an Apple product.
  • isActive (boolean): Indicates whether the product is currently active.
  • category (string): The category of the product.
  • mainImage (string): The URL of the main image of the product.
  • meanReview (number): The average review rating of the product.
  • created_at (string): The date and time when the product was created.
  • leftInStock (integer): The quantity of the product left in stock.

The response body for GET requests to the product endpoint will be in JSON format and include an array of product objects. Each product object will have the following fields:

Example 

{
  "status": "success",
  "data": [
    {
      "id": 1,
      "name": "Apple airpods pro 1",
      "description": "Good headphones for music lovers",
      "price": 150,
      "oldPrice": 0,
      "isApple": true,
      "isActive": true,
      "category": "Wireless headphones",
      "mainImage": "/media/Images/apple_airpods_pro.png",
      "meanReview": 4.2,
      "created_at": "2023-07-17T12:21:27.309298Z",
      "leftInStock": 6
    },
    ...
    ]
}

Description:
Fetches a single product by its ID. See the notes on individual parameters below.

Response
The response will be a JSON object representing the product with the following parameters:

  • id (integer): The unique identifier of the product.
  • name (string): The name of the product.
  • description (string): The description of the product.
  • price (number): The price of the product.
  • oldPrice (number): The previous price of the product (if applicable).
  • isApple (boolean): Indicates whether the product is an Apple product.
  • isActive (boolean): Indicates whether the product is currently active.
  • category (string): The category of the product.
  • mainImage (string): The URL of the main image of the product.
  • meanReview (number): The average review rating of the product.
  • created_at (string): The date and time when the product was created.
  • leftInStock (integer): The quantity of the product left in stock.

The response body for GET requests to the product endpoint will be in JSON format and include a single product object.

Example 

{
    "status": "success",
    "data": {
        "id": 1,
        "name": "Apple airpods pro 1",
        "description": "Good headphones for music lovers",
        "price": 150,
        "oldPrice": 0,
        "isApple": true,
        "isActive": true,
        "category": "Wireless headphones",
        "mainImage": "/media/Images/apple_airpods_pro.png",
        "meanReview": 4.2,
        "created_at": "2023-07-17T12:21:27.309298Z",
        "leftInStock": 6
    }
}

Description:

GET method
  • id (number): Unique identifier of each review.
  • productId (number): Unique identifier for the reviewed product.
  • userId (number): Unique identifier of the user who left the review.
  • review (number): The rating or review score left by the user.

The methods available for this endpoint are:
  • GET: Retrieves reviews.
  • POST: Adds a new review.
  • PATCH: Updates an existing review.
  • DELETE: Deletes a review.

For POST and PATCH requests, the request body must include:
  • productId (number): Unique identifier for the reviewed product.
  • userId (number): Unique identifier of the user who left the review.
  • review (number): The rating or review score left by the user.

The response format for successful requests includes a JSON object with:
  • status (string): Indicates the success or failure of the operation.
  • data (object/array): Contains the updated or newly created review object(s).
